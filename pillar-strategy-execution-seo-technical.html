Pillar GTM Playbook: Execution Chapter 2 (Technical SEO and LLM)

Joint dev and marketing implementation requirements for schema, metadata, authoring governance, and LLM files.

Execution Chapter 2: Technical SEO, Schema, Authoring & LLM Discovery

Marketing + Dev Joint Plan

1. Required Schema Entity Stack

Schema Type Required Properties Applies To
Organization name, url, logo, sameAs, contactPoint, areaServed, description Homepage, platform-level pages
WebSite SearchAction Sitewide template
Product / Service / Offer name, description, brand, offers, price, priceCurrency, eligibleRegion Pricing, features, platform pages
FAQPage Question and Answer text must match visible page copy exactly Pricing, features, demo page
Article + Person + BreadcrumbList headline, author, datePublished, dateModified, mainEntityOfPage, author profile fields All guides/blog content

2. Page Mapping Minimum

  • Homepage: Organization, WebSite, Product/Service overview.
  • Pricing page: Product + Offer per tier + FAQPage + BreadcrumbList.
  • Features page: Product/Service + ItemList + FAQPage (if visible).
  • Book demo page: Service + FAQPage + BreadcrumbList.
  • Article pages: Article + Person + BreadcrumbList + editorial metadata.

3. Authoring and Metadata Governance

  • Every indexable article must have a named author linked to an author profile page.
  • All author pages must include Person entity fields and expertise scope.
  • All articles must show publish and modified dates visibly and in metadata.
  • Minimum head tags: canonical, OG title/description/image, X card equivalents.

4. LLM Discovery Files

Root files required: /llms.txt and /llms-full.txt, maintained monthly with owner, version, and last-updated fields.
  • llms.txt: Canonical high-value URLs and usage statement.
  • llms-full.txt: Machine-readable summaries of core solution pages, FAQs, and guides.
  • Governance: Align robots, sitemap, and canonical policies with LLM file URL inventory.
