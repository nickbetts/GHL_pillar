1. Lead Scoring

Calculates ICP score from contact data and writes updates back to GHL using configurable brand profiles.

Related Webhooks

ContactCreate ContactUpdate

Exact Steps

  • Receive ContactCreate or ContactUpdate at /api/webhooks.
  • Fetch full contact via getContact(contactId).
  • Evaluate all 12 rules from lib/leadScoring.js and sum matched points.
  • Cap score at 100 using Math.min(totalScore, 100).
  • Only update contact when new score differs from current ICP Score.
  • Return matched rules in webhook response for auditability.

Actual Scoring Rules

  • Annual Income Band = "£1m+" -> +25
  • Annual Income Band = "£250k–£1m" -> +15
  • Decision Timeline = "Immediate" -> +20
  • Decision Timeline = "1–3 months" -> +15
  • Budget Authority = "Yes" -> +20
  • Primary Pain Points contains "Compliance" or "Reporting" -> +10
  • Existing Customer = "Yes" -> +30
  • Expansion Potential = "High" -> +15
  • Content Downloaded is non-empty -> +10
  • Event Attended is non-empty -> +15
  • LinkedIn Profile URL is non-empty -> +5
  • Brand profile strategic-sector match = true -> +5
Implementation source: lib/leadScoring.js. Multiple rules can stack; total is capped at 100. Keep rule sets brand-specific to avoid cross-brand taxonomy drift.
