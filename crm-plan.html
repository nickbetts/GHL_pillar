Executive Summary

This implementation runs Pillar CRM on Vercel with real-time webhook automations from GoHighLevel. Contact events trigger lead scoring, smart tagging, opportunity generation, assignment routing, and optional Apollo enrichment. The website root now hosts both strategy pages and this CRM plan while preserving all existing API routes.

  • 25 contact fields, 4 opportunity fields, 49 tag taxonomy.
  • 10-stage sales pipeline and demo booking calendar.
  • 9 automation modules with live webhook orchestration.

Architecture

Vercel serves static pages from root and serverless handlers from /api/*. The webhook endpoint is the control-plane entrypoint for all real-time CRM automation.

Contact Form -> GHL -> /api/webhooks -> calculate ICP score -> apply smart tags -> create/update opportunity -> assign owner -> optional Apollo enrichment -> write back to GHL -> sales team action queue

9 Automations

1. Lead Scoring

12 rules calculate ICP Score (0-100) using revenue, timeline, budget authority, and engagement signals.

2. Smart Tagging

8 rules auto-apply persona, intent, sector, nurture, and campaign tags in real time.

3. Opportunity Creation

Creates opportunities when high-intent or expansion conditions are met.

4. Assignment Router

Role-aware routing for fundraising, operations, enterprise, and default owners.

5. Inbound Handler

Updates intent and opt-out status from inbound SMS/email content patterns.

6. Reporting

Daily and weekly metric aggregation for pipeline and conversion visibility.

7. Apollo Enrichment

Optional person/company enrichment with ICP score bonus logic.

8. Deduplication

Exact + fuzzy duplicate detection with merge operations and audit trail.

9. Webhook Orchestration

Main event router handling Contact, Opportunity, and InboundMessage events.

Automation Mechanics (Detailed)

  • Lead Scoring: Triggered from ContactCreate and ContactUpdate. The engine evaluates rule groups (firmographic fit, urgency, authority, behavior), writes ICP Score, and only updates GHL when score changed.
  • Smart Tagging: Runs after scoring and compares computed tags against existing tags. It appends missing tags only, keeping tag operations idempotent and preventing duplicate writes.
  • Opportunity Creation: Fires when high-intent is newly introduced by smart tagging. It checks for existing opportunities first to avoid double-create, then seeds opportunity fields from contact data.
  • Assignment Router: Triggered on OpportunityCreate/Lead stage. It loads the related contact, resolves persona, then maps to environment-configured owner IDs for role-based queueing.
  • Inbound Handler: Processes inbound message text for booking and STOP intent. Booking phrases add demo-scheduled and remove high-intent to exit nurture. STOP flags communication opt-out.
  • Reporting: Daily/weekly rollups query contacts and opportunities (API-safe limits), then compute pipeline and conversion indicators for operational visibility.
  • Apollo Enrichment: Optional POST-driven enrichment flow that maps external person/org attributes to CRM custom fields and applies controlled ICP score boosts.
  • Deduplication: Uses exact email/phone and fuzzy name+company matching. Merge actions preserve notes and tag history while marking secondary records for auditability.
  • Webhook Orchestration: Central router at /api/webhooks dispatches by type and applies isolated handlers with per-event logging and safe error boundaries.
