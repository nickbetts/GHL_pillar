Executive Summary

This implementation runs Pillar CRM on Vercel with real-time webhook automations from GoHighLevel. Contact events trigger lead scoring, smart tagging, opportunity generation, assignment routing, and optional Apollo enrichment. The website root now hosts both strategy pages and this CRM plan while preserving all existing API routes.

  • 25 contact fields, 4 opportunity fields, 49 tag taxonomy.
  • 10-stage sales pipeline and demo booking calendar.
  • 9 automation modules with live webhook orchestration.

Architecture

Vercel serves static pages from root and serverless handlers from /api/*. The webhook endpoint is the control-plane entrypoint for all real-time CRM automation.

Contact Form -> GHL -> /api/webhooks -> calculate ICP score -> apply smart tags -> create/update opportunity -> assign owner -> optional Apollo enrichment -> write back to GHL -> sales team action queue

9 Automations

1. Lead Scoring

12 rules calculate ICP Score (0-100) using revenue, timeline, budget authority, and engagement signals.

2. Smart Tagging

8 rules auto-apply persona, intent, sector, nurture, and campaign tags in real time.

3. Opportunity Creation

Creates opportunities when high-intent or expansion conditions are met.

4. Assignment Router

Role-aware routing for fundraising, operations, enterprise, and default owners.

5. Inbound Handler

Updates intent and opt-out status from inbound SMS/email content patterns.

6. Reporting

Daily and weekly metric aggregation for pipeline and conversion visibility.

7. Apollo Enrichment

Optional person/company enrichment with ICP score bonus logic.

8. Deduplication

Exact + fuzzy duplicate detection with merge operations and audit trail.

9. Webhook Orchestration

Main event router handling Contact, Opportunity, and InboundMessage events.

CRM Data Model

  • Contact Fields: 25 total (identity, nurture, attribution, social proof, segmentation).
  • Opportunity Fields: 4 total (source channel, persona, org size, competitor signal).
  • Tags: 49 total across persona, funnel stage, lead source, nurture, and trust campaigns.
  • Pipeline: 10 stages from Lead to Won, Expansion, and Churn Risk.

Deployment

  • GitHub repo auto-deploys to Vercel on push to main.
  • Root URL serves static hub and plan/strategy pages.
  • API endpoints remain at /api/webhooks, /api/reports, /api/dedup, /api/apollo.
If webhook events fail, confirm environment variables are present in Vercel and validate GHL webhook URL targets /api/webhooks.

Environment Variables

  • GHL_PIPELINE_ID

  • GHL_FUNDRAISING_OWNER
  • GHL_OPS_OWNER
