4. Assignment Router

Routes opportunities to the right owner based on persona and stage.

Related Webhooks

OpportunityCreateOpportunityUpdate

Exact Steps

  • OpportunityCreate event is received by webhook router.
  • If stage is Lead and owner is empty, contact is fetched for role context.
  • Persona is mapped to role-specific owner env vars.
  • Fallback owner is used if no specific mapping exists.
  • Opportunity is updated with selected owner ID.
  • Assignment action is logged for operations visibility.
Owner env vars must contain valid GHL user IDs, not display names.
