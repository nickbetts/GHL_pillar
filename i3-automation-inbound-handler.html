5. Inbound Handler

Interprets inbound message intent and updates CRM state immediately.

Related Webhooks

InboundMessage

Exact Steps

  • Read contactId and message body from InboundMessage payload.
  • Scan lowercase body for booking terms: book, schedule, or demo.
  • If booking intent found: add demo-scheduled and remove high-intent.
  • Scan uppercase body for STOP keyword.
  • If STOP found: add opted-out-sms.
  • Update contact only when tags changed.

Actual Logic Rules

  • Booking branch condition: body includes "book" OR "schedule" OR "demo" (case-insensitive).
  • Booking branch writes: tags + demo-scheduled, then removes high-intent.
  • Suppression condition: body includes "STOP" (uppercase check on transformed message).
  • Suppression writes: tags + opted-out-sms.
  • Both branches can execute in the same message if both conditions match.
