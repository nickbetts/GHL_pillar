2. Smart Tagging

Applies dynamic tags from persona, segment, engagement, and pain-point signals using brand-specific taxonomies.

Related Webhooks

ContactCreate ContactUpdate

Exact Steps

  • Load existing tags from contact.tags.
  • Evaluate all 8 tagging rules in lib/smartTagging.js.
  • Add only net-new tags (no duplicate tag writes).
  • Persist updated tag list only when additions exist.
  • If high-intent is newly added, pass control to opportunity creation logic.

Actual Tagging Rules

  • Annual Income Band = "£1m+" and Persona = "Decision Owner" -> decision-owner, org-500plus, high-intent
  • Primary Pain Points contains "Spreadsheets" or "Tool sprawl" -> high-intent, consideration-stage
  • Strategic Segment = "Core Segment" -> core-segment
  • Content Downloaded non-empty OR Event Attended non-empty -> retargeting-warm, consideration-stage
  • Form Abandonment Flag = "Yes" -> retargeting-warm
  • Existing Customer = "Yes" and Expansion Potential = "High" -> expansion, customer
  • Case Study Candidate = "Yes" and Annual Income Band = "£250k–£1m" -> case-study-candidate, trust-sprint
  • Seasonal Campaign Relevance = "Yes" and Strategic Segment = "Core Segment" -> seasonal-campaign, calendar-relevant
Implementation source: lib/smartTagging.js. Tagging is idempotent by design. Configure persona/segment tags per brand profile before enabling in production.
