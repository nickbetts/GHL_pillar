5. Inbound Handler

Interprets inbound message intent and updates CRM state immediately.

Related Webhooks

InboundMessage

Exact Steps

  • Inbound message event arrives with contact ID and message body.
  • Message content is scanned for booking intent terms.
  • If booking intent is detected, demo-scheduled tag is added and high-intent is removed.
  • Message content is scanned for STOP keyword.
  • If STOP is detected, opted-out-sms tag is added for suppression logic.
  • Contact tags are written back to GHL and event is logged.
This handler is the fastest path from lead reply to nurture-state change.
