i3MEDIA GTM Playbook: Paid Meta Retargeting

CAPI-backed warm-audience recapture with high-variation creative deployment.

Retargeting/Research | Paid Meta Retargeting (CAPI Engine)

High-Frequency Validation

1. Conversions API (CAPI) Architecture Backend

Enforce a modern tracking setup to protect custom remarketing pools from browser cookie drops:

  • Server-to-Server Integration: Deploy Meta Conversions API (CAPI) linked directly to the database layer, matching work email hash identifiers to secure high Event Match Quality indicators.
  • Exclusion Protocol Rules: Apply live exclusions to active customer profiles and converted calendar bookings to protect daily ad spend.
  • UK Location Fee Surcharge Warning: Account for the mandatory 2% UK Location Surcharge fee parameter directly inside line-item asset models to neutralise balance errors.

Meta's delivery engine prioritises ad variety and visual targeting. Deploy six distinct creative tracks with video-first emphasis:

Build form, set recurring, add email, attach Gift Aid, publish → Stripe syncs → 47 donors in 24h

3. Campaign and Ad Set Structure

Layer Audience Logic Creative Mix Primary Objective
Warm Intent Retargeting High-intent page viewers and form starters Concepts 1, 2 (Tool Sprawl, Speed Demo) Booked demo efficiency
Mid-Funnel Re-engagement Content engagers and repeat site visitors Concepts 3, 4, 5 (Gift Aid, Donor Experience, Retention) Qualified return traffic and lead intent
Reactivation Longer-window visitors with no conversion Concepts 5, 6 (Retention Audit, Regret Testimonial) Recover stalled opportunities

4. Event and Signal Framework

  • Core events: form start, form submit, booked demo, and qualified meeting attended.
  • Deduplication standard: use consistent event IDs across browser and server streams.
  • Source taxonomy: retain campaign, ad set, and creative identifiers through CRM sync for downstream pipeline attribution.
  • Suppression signal: once a demo is booked, remove contacts from active retargeting within one sync cycle.

5. Creative Rotation and Fatigue Controls

Rotation Rules

  • Maintain at least three concept families live at the same time.
  • Refresh weakest variants every 10-14 days once stable delivery is established.
  • Keep one challenger creative per concept family active continuously.

Fatigue Rules

  • Watch rising frequency with falling qualified click-through trend.
  • Replace repetitive hooks before delivery narrows to one message family.
  • Preserve format diversity across image, video, and carousel placements.

6. Performance Review Cadence

  • Twice weekly: check CAPI health, event match quality trends, and audience suppression lag.
  • Weekly: evaluate booked-demo CPA and meeting quality by audience and concept family.
  • Monthly: rebalance budget toward top-performing audience windows and retire chronically weak concept families.
